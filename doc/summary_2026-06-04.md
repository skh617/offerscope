## 2. 遇到的问题与解决方案

### 2.1 飞书卡片链接无法打开报告

| 尝试 | 结果 | 原因 |
|------|------|------|
| `file://` 链接 | 点击无反应 | 飞书客户端不支持 file 协议打开本地文件 |
| `http://localhost:8091/中文文件名.html` | 白屏 | 飞书打开浏览器时对中文 URL 编码错乱 |
| `http://localhost:8092/report/ai` | ✅ 成功 | ASCII 短链接，serve.py 路由层映射到实际文件 |

**最终方案**：`serve.py` 提供 `/report/{slug}` 短链接路由，内部匹配最新报告文件。

### 2.2 飞书关键词验证

飞书机器人开启了安全设置（关键词验证），卡片内容必须包含设定的关键词 `offer`。在 `build_card()` 末尾追加验证文本解决。

### 2.3 端口占用

旧服务进程未正常退出导致端口被占用，`cmd //c "taskkill /F /PID xxx"` 杀掉后重启解决。

### 2.4 Windows 终端 GBK 编码

终端无法显示 emoji 和中文，`UnicodeEncodeError`。改用 `[+]` / `[-]` 替代 🟢/🔴，print 包裹 try/except 处理。

## 4. 当前使用方式

```bash
# 1. 启动报告服务（保持后台运行）
python serve.py

# 2. 编辑 config.yaml 配置抓取任务

# 3. 执行全流程（浏览器扫码登录后自动抓取→分析→推送）
python scheduler.py

# 4. 仅预览（使用已有数据，不抓取不推送）
python scheduler.py --dry-run
```

**飞书卡片链接**：`http://localhost:8092/report/ai`

## 5. 后续优化建议

1. **定时调度**：APScheduler / cron 定时执行（v1.1 规划）
2. **多城市并行**：同一关键词多城市一次性抓取对比
3. **趋势数据库**：多次抓取后环比数据更有参考价值
4. **手机端访问**：局域网 IP 替代 localhost，手机也能看报告
5. **serve.py 常驻**：配置为 Windows 服务或开机自启
