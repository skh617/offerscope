# 04 — AI 相似岗位扩展

> 状态：规划中 | 优先级：P1（后续版本） | 依赖：无 | 2026-06-07

## 问题

当前搜索只使用用户输入的单一关键词，不做近义词扩展。搜 "AI应用后端工程师" 只会命中标题包含这些词的岗位，可能漏掉 "AI开发工程师"、"大模型开发"、"AIGC开发" 等相关岗位。

## 方向

**不用预设词典做关键词映射**（维护成本高、覆盖面窄），而是通过 AI/LLM 动态分析输入的关键词，生成语义相近的搜索词列表。

## 大致思路

1. 用户输入岗位名（如 "AI应用后端工程师"）
2. 调用 LLM 分析，生成 3-5 个近义搜索词（如 "AI开发工程师"、"大模型应用开发"、"AIGC后端"）
3. 每个搜索词单独抓取，结果合并去重（`collect_list` 已支持去重）
4. 所有结果标记来源关键词，报告中展示覆盖范围

## 待定

- 用哪个模型（OpenAI / Claude API / 本地模型）
- 是否做成可选开关（避免不必要的 API 调用成本）
- 搜索词数量上限（防止 API 调用和抓取时间膨胀）
