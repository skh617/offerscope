## 3. Web 化架构

```
远程用户浏览器                 服务器（本地 PC）
┌──────────────┐    HTTP     ┌──────────────────────┐     ┌─────────────┐
│ 前端单页 HTML │◄──────────►│  web_app.py (FastAPI) │────►│ CloakBrowser│
│ - 表单        │            │  GET  /               │     │ (有头模式)  │
│ - QR 码展示   │            │  POST /api/scrape     │     │ 持久化 Profile│
│ - 进度轮询    │            │  GET  /api/status/:id │     └──────┬──────┘
│ - 报告展示    │            │  GET  /report/:id     │            │
│ - 历史列表    │            │  GET  /api/reports    │      BOSS直聘
└──────────────┘            └──────────────────────┘
                                    │
                            Cloudflare Tunnel
                            (内网穿透 → HTTPS)
```

### API 路由

| 方法 | 路径 | 功能 |
|------|------|------|
| `GET` | `/` | 返回前端页面 |
| `POST` | `/api/scrape` | 创建抓取任务 `{keyword, city?, max_jobs?, webhook?}` |
| `GET` | `/api/status/{job_id}` | 查询任务状态（含 `qrcode` / `qrcode_url`） |
| `GET` | `/report/{job_id}` | 查看 HTML 报告 |
| `GET` | `/api/reports` | 最近报告列表（扫描 reports/ 目录） |

### 任务状态模型

```
pending → running → done
                 → error
```

JobState 字段：`job_id`, `status`, `progress`, `qrcode`, `qrcode_url`, `keyword`, `city`, `jobs_count`, `report_path`, `webhook_sent`, `error`, `created_at`

## 4. 关键技术突破：远程扫码登录

### 4.1 问题演进

| 阶段 | 问题 | 解决方案 |
|------|------|----------|
| 初始 | 浏览器在服务器上，远程用户看不到登录页 | 截图页面传 base64 |
| 优化 1 | CSS 选择器匹配到错误元素 | 改为截整个 viewport 页面 |
| 优化 2 | 截图比例被压缩、分辨率低无法扫码 | 提取 QR 图片原始 URL 传给前端 |
| 优化 3 | `QR_URL:` 前缀被 `_qr` lambda 二次包装为 `QR:QR_URL:` | `_qr` 改为直接透传 |
| 优化 4 | 页面默认显示短信登录，非二维码 | 自动点击"微信登录"切换标签 |

### 4.2 最终数据流

```
1. scraper: page.goto(登录页)
2. 自动点击"微信登录"标签
3. 查找 iframe/img 的 QR 图片 URL → on_qr("QR_URL:https://...")
4. web_app: _set_progress 识别 QR_URL: 前缀 → job.qrcode_url
5. 前端: <img src="原始URL"> 渲染高清晰度 QR 码
6. 用户手机扫码 → cookie 出现 → login_wait 返回 True
7. 第一条非 QR 进度消息 → qrcode 自动清空 → QR 消失 → 继续抓取
```

## 5. 遇到的问题与解决方案

### 5.1 Session 未持久化（每次都要重新登录）

| 根因 | 解决方案 |
|------|----------|
| `launch()` 每次创建全新无痕浏览器 Profile，关闭时全部销毁 | 改用 `launch_persistent_context(user_data_dir="boss-profile")`，Cookie/localStorage 自动存盘 |

### 5.2 远程扫码 — QR 码错误

| 尝试 | 结果 | 原因 |
|------|------|------|
| CSS 选择器找 `.qrcode-img img` 等 | 匹配到非二维码元素 | BOSS 页面结构复杂，选择器不可靠 |
| 截整个 Viewport | QR 码太小无法扫 | 整页截图分辨率不足 |
| **提取 QR 图片原始 URL** | ✅ 成功 | 前端直接加载原始二维码图片，清晰度完美 |

### 5.3 `QR_URL:` 被错误解析为 base64

| 根因 | 解决方案 |
|------|----------|
| `_qr` lambda 自动给所有数据加 `QR:` 前缀 → `QR_URL:url` 变成 `QR:QR_URL:url` → 被匹配到 base64 分支 | `_qr` 改为直接透传，不二次包装 |

### 5.4 BOSS 登录页默认显示短信登录而非二维码

| 根因 | 解决方案 |
|------|----------|
| 页面默认 Tab 是"验证码登录" | 自动遍历选择器点击"微信登录"、"扫码登录"等标签切换 |

### 5.5 Windows GBK 终端编码崩溃

| 症状 | 解决方案 |
|------|----------|
| 岗位名/薪资含 `‌` 等特殊 Unicode → `UnicodeEncodeError` → 程序卡住 | 新增 `_safe_print()` 函数，`UnicodeEncodeError` 时自动 fallback 为 ASCII |

### 5.6 QR 码图片在页面上比例变形

| 根因 | 解决方案 |
|------|----------|
| CSS 强制 `width:180px; height:180px` | 改为 `max-width:320px; height:auto` 保持原始比例 |

### 5.7 环境依赖遗漏

| 问题 | 解决 |
|------|------|
| venv 中未安装 FastAPI/uvicorn | `pip install fastapi "uvicorn[standard]"` |

### 5.8 cloudflared 下载缓慢 / 文件锁定

| 问题 | 解决 |
|------|------|
| GitHub Releases 下载国内极慢（54MB） | 用户在浏览器中手动下载 `cloudflared-windows-amd64.exe` |

## 6. 当前使用方式

```bash
# 终端 1：启动 Web 服务
python web_app.py
# → http://localhost:8090

# 终端 2：内网穿透（分享给他人）
cloudflared tunnel --url http://localhost:8090
# → https://xxx.trycloudflare.com  ← 分享这个链接
```

### 使用流程

1. 打开链接 → 看到表单 + 最近报告列表
2. 填写岗位名（必填）、城市/数量/飞书 Webhook（选填）
3. 点击"开始分析"→ 页面显示 BOSS 登录二维码
4. 手机扫码登录（用自己的 BOSS 账号）
5. 实时查看 5 阶段抓取进度
6. 填了 webhook → 推送飞书 + 页面查看报告
7. 没填 webhook → 页面内嵌展示完整报告

### 首次 vs 再次

| 次数 | 行为 |
|------|------|
| 首次提交 | 显示 QR 码 → 扫码登录 → Session 保存到 `boss-profile/` |
| 再次提交 | 检测到有效 Session → 跳过 QR → 直接抓取 |
