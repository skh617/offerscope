# BOSS 直聘岗位追踪系统 — 需求文档 v1

> 2026-06-03 讨论沉淀

## 1. 产品概述

### 1.1 项目定位

智能岗位抓取与分析平台。帮助用户持续关注目标岗位市场的技能要求、薪资水平、经验学历门槛的变化趋势，以便查漏补缺，并通过原文链接快速与 HR 沟通。

### 1.2 核心用户场景

1. 用户配置关注的关键词 + 城市（如"Java后端开发 · 北京"）
2. 系统定期抓取 BOSS 直聘岗位数据
3. 系统分析当前市场快照，对比历史数据生成趋势
4. 通过飞书推送摘要卡片，用户快速了解市场变化
5. 用户点击详情链接查看完整报告，找到目标岗位后点击原文链接联系 HR

### 1.3 当前阶段边界

| 维度 | 范围 |
|------|------|
| 平台 | 仅 BOSS 直聘（后续扩展拉勾、猎聘等） |
| 抓取方式 | 手动触发 + 配置文件驱动（后续做定时调度 + Web 管理面板） |
| 推送渠道 | 飞书 Webhook（后续扩展钉钉、企业微信等） |
| 数据存储 | JSON 文件按日期归档（后续迁移 SQLite） |

## 2. 功能需求

### 2.1 任务配置管理

#### 2.1.1 配置文件

用户通过 YAML 配置管理抓取任务。

**配置结构：**

```yaml
tasks:
  - keyword: Java后端开发
    cities: [北京, 上海, 杭州]
    webhook: https://open.feishu.cn/open-apis/bot/v2/hook/xxx
  - keyword: AI应用开发
    cities: [北京, 深圳]
    webhook: https://open.feishu.cn/open-apis/bot/v2/hook/yyy
```

**配置项说明：**

| 字段 | 说明 |
|------|------|
| `keyword` | 搜索关键词 |
| `cities` | 目标城市列表 |
| `webhook` | 飞书机器人 Webhook URL |

后续扩展字段（暂不实现）：`schedule`（定时表达式）、`enabled`（启停开关）。

#### 2.1.2 任务执行入口

- **手动触发**：通过 Claude Code 对话执行（如"抓取岗位数据"）
- **调度脚本**：`scheduler.py` 读取 config.yaml，遍历任务依次执行
- 后续接入定时调度器（APScheduler / 系统 cron）

### 2.2 岗位抓取

复用现有 `tools/scraper.py`，不做核心逻辑改动。

- CloakBrowser 反检测浏览器
- 触底滚动翻页加载列表
- 逐个进入详情页获取薪资、JD 描述、公司信息
- 去重、限流恢复
- 输出：`data/{keyword}_{city}_{date}_jobs.json`

#### 2.2.1 登录会话管理

BOSS 直聘需要扫码登录，用户可能不在电脑前。策略如下：

**当前方案：持久化会话 + 优雅降级**

1. CloakBrowser 保存浏览器 profile（cookie/localStorage），一次扫码登录后持久化会话
2. BOSS 直聘登录态通常维持数小时，当天任务自动复用
3. 会话过期时：飞书通知用户，对应任务标记为跳过，下次重试
4. 用户回到电脑前扫码后，会话恢复，任务正常执行

**后续扩展：远程扫码**

> 记录为待实现功能，当前不做。

1. 会话过期时，系统自动打开登录页并截图二维码
2. 通过飞书发送截图给用户
3. 用户在手机上打开截图扫码登录
4. 浏览器检测到登录态恢复，继续执行任务

实现前提：飞书 Webhook 不支持发送图片，需升级为飞书企业应用（Image API）。

#### 2.2.2 抓取频率与数量

| 参数 | 推荐值 | 原因 |
|------|--------|------|
| 抓取频率 | 每周 1 次 | 岗位技能需求变化以周为单位才有统计意义；日间波动多为同岗刷新排名，非实质性市场变化；周级间隔更像正常用户行为 |
| 每任务条数 | 30-50 条 | <20 条统计不可靠（薪资中位数波动大、分词样本不足）；30-50 条各项指标稳定，高薪/入门分组有足够样本；>80 条收益递减，低相关岗位混入，详情页请求量翻倍增加封号风险 |
| 滚动轮数 | 2-3 轮 | 每轮约 15 条，2-3 轮即 30-45 条，落在适中区间 |
| 详情页间隔 | 3-8 秒随机 | 现有逻辑已合理，保持不变 |

多个关键词任务建议分散在不同时间/天执行，避免短时间内密集请求触发反爬。

### 2.3 市场分析

#### 2.3.1 分析维度复用

基于现有 `tools/analyzer.py`，复用以下分析函数：

| 函数 | 输出 |
|------|------|
| `classify_jobs()` | 按关键词分组 |
| `get_top_skills()` | jieba 分词提取技能词频 |
| `extract_salary_range()` | 薪资中位数 |
| `salary_histogram()` | 薪资区间分布 |
| `experience_stats()` | 经验要求分布 |
| `education_stats()` | 学历要求分布 |
| `select_radar_dims()` | 技能雷达图（24 维自适应选 8 维） |
| `skill_diff_high_vs_low()` | 高薪 vs 入门技能对比（分水岭技能） |

#### 2.3.2 需修改项

| 位置 | 当前 | 改为 |
|------|------|------|
| 岗位弹窗（Modal） | 展示岗名/薪资/经验/学历/公司/行业/规模/JD 描述 | 增加 BOSS 直聘原文链接 |
| 薪资概要面板 | 展示中位数/均值/P75/P25/样本数 | 删除 P75 和 P25 两行 |

### 2.4 趋势对比

新增模块 `tools/trend.py`。

#### 2.4.1 功能

读取当前抓取结果与历史 JSON（同关键词），计算环比变化。

#### 2.4.2 计算指标

| 指标 | 计算方式 |
|------|---------|
| 薪资中位数环比 | 本次中位数 - 上次中位数 |
| 技能词提及率变化 | 每个技能词本次提及率 - 上次提及率 |
| 技能上升 TOP3 | 提及率涨幅最大的 3 个技能 |
| 技能下降 TOP3 | 提及率跌幅最大的 3 个技能 |
| 高薪分水岭技能 | 复用 `skill_diff_high_vs_low()`，取高薪组独有技能 TOP3 |

#### 2.4.3 边界情况

- 首次抓取（无历史数据）：仅展示当前快照，环比标记为"首次"
- 历史数据缺失：降级展示，不阻断流程

### 2.5 飞书推送

新增模块 `tools/publisher.py`。

#### 2.5.1 推送卡片内容

| 序号 | 信息块 | 内容 |
|------|--------|------|
| 1 | 标题栏 | 关键词 + 城市 + 时间戳 + 样本数 |
| 2 | 💰 薪资中位数 | 当前值 + 环比涨跌标记 |
| 3 | 🔥 技能热度 TOP8 | 技能词列表，上涨/下跌/持平三色区分 |
| 4 | 💎 高薪分水岭技能 | 高薪岗位独有、入门岗少见的技能 2-3 个 |
| 5 | 📈 技能热度变化 | 上升 TOP3（绿色）+ 下降 TOP3（红色） |
| 6 | 📋 门槛概况 | 主流经验 + 主流学历（各自最高占比区间） |
| 7 | 📊 详情链接 | 跳转完整 HTML 分析报告 |

#### 2.5.2 推送策略

- 每次抓取完成后推送一次
- 推送目标读取 config.yaml 中的 webhook 地址
- 失败时记录日志，不重试（后续版本加告警）

#### 2.5.3 不推送内容

以下内容仅在完整报告中呈现：
- 技能雷达图
- 薪资区间柱状图
- 经验/学历环形图
- 高薪 vs 入门技能对比详表
- 完整岗位明细表（含分页 + 原文链接）

### 2.6 完整报告

基于现有 `tools/analyzer.py` 生成的 HTML 报告，包含 10 个模块：

1. **岗位画像卡片** — 按关键词分组概览
2. **技能雷达图** — Chart.js，24 维自适应选 8 维
3. **技能词频 TOP15** — jieba 自动提取
4. **薪资区间分布** — 7 档柱状图
5. **薪资概要** — 中位数、均值、样本数
6. **经验要求分布** — 环形图
7. **学历要求分布** — 环形图
8. **高薪 vs 入门技能对比** — 两组各 5 个
9. **市场信号** — 文字总结
10. **岗位明细表** — JS 分页表格 + 点击弹窗（含原文链接）

## 3. 系统架构

### 3.1 文件结构

```
offerscope/
├── config.yaml                  # 新增：任务配置
├── scheduler.py                 # 新增：读取配置 → 串联流程
├── tools/
│   ├── scraper.py               # 不改：抓取岗位
│   ├── analyzer.py              # 小改：弹窗加链接 + 删除 P75/P25
│   ├── trend.py                 # 新增：趋势对比
│   └── publisher.py             # 新增：飞书推送
├── data/                        # JSON 岗位数据 + 历史快照
└── reports/                     # HTML 分析报告
```

### 3.2 执行流程

```
config.yaml
    │
    ▼
scheduler.py ─── 遍历任务
    │
    ▼
scraper.py ─── 抓取 → data/{keyword}_jobs.json
    │
    ▼
trend.py ─── 读取 本次 JSON + 历史 JSON → 计算环比
    │
    ▼
analyzer.py ─── 生成完整 HTML 报告 → reports/
    │
    ▼
publisher.py ─── 组装卡片 → POST 飞书 Webhook
```

## 4. 非功能需求

### 4.1 模块独立性

- `scraper.py`、`analyzer.py`、`trend.py`、`publisher.py` 均通过 JSON 数据文件解耦
- 每个模块可独立运行和测试
- 单点失效不阻断整体流程（如趋势计算失败，仍可推送当前快照）

### 4.2 兼容性

- 新增模块不应修改 `scraper.py` 和 `analyzer.py` 的核心逻辑
- analyzer.py 仅做界面小改（弹窗加链接、删除 P75/P25）

### 4.3 扩展预留

- 配置结构预留 `schedule`、`enabled` 字段
- 推送模块预留多渠道扩展接口（webhook 抽象）
- 数据目录结构预留日期归档命名规范

## 6. 术语表

| 术语 | 说明 |
|------|------|
| 采样快照 | 一次抓取获得的所有岗位数据，视为该时刻的市场横截面 |
| 环比 | 本次快照与上一次快照之间的数值变化 |
| 高薪分水岭技能 | 高薪岗位（P75+）中高频出现、入门岗位（P25-）中少见的技能词 |
| 技能提及率 | 某一技能词在 JD 描述中出现的岗位数 / 总岗位数 |
| 主流经验/学历 | 所有岗位中占比最高的经验/学历区间 |
